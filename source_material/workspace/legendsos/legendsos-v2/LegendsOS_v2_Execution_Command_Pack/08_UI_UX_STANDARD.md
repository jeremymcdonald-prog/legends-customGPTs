# UI and UX Standard

## Goal

LegendsOS v2.0 must look and feel like one premium internal operating system.

No stitched together subdomains.

No raw third party pages as primary experience.

No clutter.

No fake modules.

## Visual style

- dark cinematic background
- black and navy base
- gold or orange accents
- white text
- clean cards
- soft borders
- strong hierarchy
- premium but practical
- fast loading

## Navigation

One sidebar and one top header.

Suggested sections:

- Command Center
- Atlas
- Social Studio
- Images
- Email
- Calendar
- Admin
- Settings

Only show Admin to Jeremy or admins.

## Role aware UI

Regular loan officer:

- own Atlas assistant
- shared Social Studio
- shared Image Studio
- own Email Studio
- Calendar
- Settings

Jeremy:

- all modules
- admin
- team usage
- provider settings
- team user management

## What not to show

- other individual loan officer assistants to non owners
- Vivian Delgado
- old v1 modules
- borrower portal
- realtor portal
- mortgage ops
- loan flow
- blog studio
- raw Postiz
- Onyx branding
- empty side rails
- duplicate nav
- login loops
- hidden broken content

## Empty states

Every empty state must be useful:

Wrong:

- blank page
- no agents found
- raw error
- network error

Right:

- no drafts yet, create one
- provider missing, add key in Settings
- no scheduled posts, create social post
- upload not configured, contact admin

## Mobile

Minimum support:

- sidebar collapses
- cards stack cleanly
- chat usable
- social draft usable
- image generation form usable
