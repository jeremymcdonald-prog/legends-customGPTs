# Week 1 Production Runner Spec

**Purpose.** Define the runnable workflow that consumes the Week 1 prompt and tracker files and produces ready media without operator busywork. Replaces the prior manual-step report.

## Input files
- `04_prompt_banks/week_1_canva_batch_prompts.csv`
- `04_prompt_banks/week_1_heygen_batch_prompts.csv`
- `04_prompt_banks/week_1_higgsfield_batch_prompts.csv`
- `04_prompt_banks/week_1_thumbnail_prompts.csv`
- `05_assets/week_1_asset_generation_plan.csv`
- `06_heropost/week_1_heropost_staging_locked.csv`
- `05_assets/media_url_tracker.csv`

## Runner steps

### Step 1. Read all Week 1 asset rows
Load the asset generation plan. Confirm 58 rows. Index by `(day, platform, template_id)`.

### Step 2. Group by tool and template
Bucket assets into 7 operator groups via the operator-group map:
- Feed static (IG_FEED, FB_BUSINESS, FB_PERSONAL with template)
- GBP static
- LinkedIn static
- Story static
- Reel cover
- YouTube thumbnail (Short + Long)
- TikTok cover (reuses Reel cover template)
Plus 1 HeyGen short bucket and 1 Higgsfield plate bucket.

### Step 3. Canva
Programmatic autofill requires the Canva-side brand template IDs to be paired with our local template IDs first. Until that mapping exists, the runner outputs **one Bulk Create CSV per template group** (this sprint creates those CSVs). When mapping is captured, the runner can switch to `mcp__f62fe56b-...__autofill-design` calls per row.

### Step 4. HeyGen
Submit exactly **one** test render first using `week_1_heygen_test_payload.json`. Poll status. If pass, queue the remaining 18 from `week_1_heygen_batch_after_test_packet.md` (now driven by the operator master queue). If fail, hold the batch; flag in the dispatch tracker.

### Step 5. Higgsfield
No connector available. Output **one consolidated copy-paste batch file** containing all 3 prompts + negative prompts + filenames + destinations. Operator runs in Higgsfield UI.

### Step 6. Drive
Create the folder structure under jeremy@mcdonald-mtg.com:
```
Drive: LegendsOS_60Day/Week_01/exports/
  Feed_static/
  GBP_static/
  LinkedIn_static/
  Story_static/
  Reel_cover/
  YouTube_thumbnail/
  TikTok_cover/
  HeyGen/test/
  HeyGen/batch/
  Higgsfield/
```
Upload exports from each tool to the matching subfolder. The export-upload-map.csv already tracks destinations.

### Step 7. Media URLs
Update `05_assets/media_url_tracker.csv` ONLY when a real `https://...` URL is generated by Drive sharing. Status moves to `ready`. Runner never invents URLs.

### Step 8. HeroPost
Promote staging rows to a final CSV ONLY when:
- `Media` resolves to a real URL.
- `Thumbnail` resolves to a real URL (for video rows).
- All 8 gates in `06_heropost/week_1_final_promotion_rules.md` pass.
Output: `06_heropost/heropost_week_1_ready.csv` (created only when rows actually qualify).

## Hard rules
- No fabricated URLs.
- No final CSV created without real URLs.
- No publishing.
- No HeroPost upload from this runner.

## Automation handoff
Once Canva brand template IDs are paired, steps 3 and 6 collapse into one n8n workflow:
1. Read prompt CSVs
2. Call Canva autofill-design per row
3. Wait for design completion event
4. Call export-design
5. Upload export to Drive folder
6. Read Drive shareable URL
7. Write to media_url_tracker.csv
8. Trigger HeroPost Scheduler promotion

## Current runner output
This sprint runs steps 1, 2, 3 (CSV output mode), 5, and 6 (spec-only). Steps 4, 7, 8 are operator-gated until HeyGen test passes and real exports land.
