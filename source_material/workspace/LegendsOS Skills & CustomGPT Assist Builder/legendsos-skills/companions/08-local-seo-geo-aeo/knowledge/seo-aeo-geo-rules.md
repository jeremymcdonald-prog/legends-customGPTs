# SEO, AEO, GEO Rules

Three layers, one strategy. SEO for search engines. AEO for answer engines (Google AI Overviews, ChatGPT, Claude, Perplexity, Gemini). GEO for the local generative answer surface.

## SEO basics (still required)

- Clean H1 per page.
- One target keyword per page, natural usage.
- Meta title 50 to 60 characters.
- Meta description 140 to 160 characters.
- Alt text on every image with the local angle baked in (never stuffed).
- Mobile-first layout.
- Page speed budget under 2.5 seconds LCP.
- Internal links 2 to 4 per page.
- External links to authoritative sources where stats are used.

## AEO

- Lead with a direct answer in the first 1 to 2 sentences.
- Use Q-and-A formatted H2s where appropriate.
- Use FAQPage schema when the page is genuinely Q-and-A.
- Cite primary sources for any number, statistic, or program rule.
- Plain English. AI answer engines reward the cleanest writing.
- Avoid clickbait headlines that don't pay off in the first paragraph.

## GEO (Generative Engine Optimization)

- Authority signals matter more than ever: NAP consistency, real reviews, in-market mentions, structured data.
- Be the source AI cites. Long form on mcdonald-mtg.com should be the canonical version of any topic; social pieces should derive from it.
- Local context: name the city, name the county, name the neighborhood naturally. Generative answers favor specific over generic.
- Author entity: every piece attributes Jeremy McDonald as the author with NMLS context. AI answer engines weight named expert authorship.

## Schema recommendations by page type

| Page type | Primary schema |
|---|---|
| Home | LocalBusiness + WebSite |
| Service page (VA, FHA, FTHB, DSCR) | Service + LocalBusiness + FAQPage |
| Blog post | Article |
| Vlog page | VideoObject + Article |
| FAQ page | FAQPage |
| About / team page | Person + Organization |
| Testimonial page | Review (only with public, sourced reviews) |
| Calculator | SoftwareApplication or WebPage with calculator markup |

## Internal link strategy

- Service pages link to each other where products overlap (VA <-> FTHB, DSCR <-> Investor).
- Blog posts link to relevant service pages.
- Calculators link back to the relevant service page.
- Author bio links to all service pages.
- Footer carries NAP, NMLS, EHO, primary service area, sitemap link.

## Anti-patterns

- Doorway pages (one page per zip code with the same body).
- Hidden text or invisible keyword lists.
- Reciprocal link schemes.
- Cloaking.
- AI-generated mass content without a real author voice or value.
